    1. Incorporating Local Versions of Packages

    Edit the local.mk file in the buildroot-2017.02.04 directory and uncomment the package you wish to include a local version of. When you run make again the source code in your local directory will be packaged up and incorporated as part of the build.

    Tip

    You can see what packages currently have been built locally by using the following command 

    ls output/build/ | grep custom

    One thing to be aware of is if you have not cleaned the package in a while there still may be a custom package left over from previous builds.

    1.1 Cleaning Package

    Removing a package is unsupported by Buildroot without rebuilding from scratch. This is because Buildroot doesnt keep track of which package installs what files in the output/staging and output/target directories, or which package would be compiled differently depending on the availability of another package.

    The easiest way to rebuild a single package from scratch is to remove its build directory in output/build. Buildroot will then re-extract, re-configure, re-compile and re-install this package from scratch. There is a helper script available to do this which can be called as follows

    ./run clean-package <Space delimited list of package names>

#e.g.

./run clean-package server_emerald capture-layer libsystem

    1.2 Updating local.mk

    You can update this file when you add a new package by running autocreate_local.py. This will produce the output to standard out which you can then pipe to a file. In the buildroot-2017.02 directory call.

    ./autocreate_local.py > local.mk

    Tip

    To prevent accidental commiting of the local.mk file a pre commit hook blocks committing by anyone but the SVN administrator.
