    The following functionality is supported in this release

    • System auto boots to the OSD
    • System can operate in unmanaged mode or managed mode (Managed mode is limited to making connections to managed appliances only, there is no Boxilla configuration or provisioning of connections and users)
    • A user with appropriate permissions can create connections and workspaces in an unmanaged SDKVM device.
    • The SDKVM can be discovered by Boxilla.
    • The SDKVM can be managed by Boxilla. (but does not support Boxilla provisioned connections). It will copy the XML file (but not use it), place a Boxilla token on SDKVM, change IP address to a static IP.
    • Boxilla can reboot the SDKVM over REST
    • Boxilla can retrieve Product Details over REST
    • Boxilla can manually manage a SDKVM without having to depend on the discovery protocol.
    • The SDKVM can respond to a Boxilla Ping
    • The system will support multiple users in unmanaged mode and each of these users will be able to maintain a separate connection and workspace.
    • It will be possible to configure and persist network settings through the OSD allowing the user set a static or DHCP address. (Note: Boxilla does not support DHCP).
    • Hotkeys are managed locally but there functionality will be limited to terminating connections. (Launching workspaces through a hot key will not be supported)
    • A user can launch workspaces based on pre-provisioned connections from the OSD workspace screen.
    • The OSD reflects the status of any current workspaces. If the workspace is terminated by a hot key then the OSD will update to reflect that change in status. The connect button will offer a termination option once the workspace is active.
    • Boxilla can retrieve information for a version compatibility check.
    • If a connection fails the tile for that connection should provide a clear failure message in that tile space.

    Required

    Boxilla version BXA 4.7.0 EC1 build is required for testing

    Not Supported

    • No audio support in this release.
    • Configuration of users and connection on the SDKVM through Boxilla is not supported.
    • It is not possible to launch Workspaces from hot keys

    Known Issues

    • The OSD performance is very poor
    • The monitor detection and configuration logic is not working reliably from the OSD
    • Hot key mapping and hot key binding is not functional
    • Connections to GE are not functional
    • KVM connections are prone to freezing
    • KVM connections will occasionally terminate without notice if left in a soak
    • The OSD does not always reflect the workspace status in a timely manner
    • The underlying OS is exposed by right clicking on the desktop
    • Discovery service mail fail on boot up
    • Disconnecting and reconnecting the network cable will break cause the discovery service to malfunction
    • DNS for connections is not working due to an inability to set DNS on the network interfaces page.
