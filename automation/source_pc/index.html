    1. Source PC Automation

    As part of automating testing of Transmitter and Receiver devices it must be possible to remotely control the source PC connected to the Transmitter. To achieve this the proposal is to run a REST server on the source PCs to invoke the functionality needed. This application will be called the Source PC Automation Server.

    1.1 Setting Up A Target PC

    1.1.1 NoMachine

    NoMachine is a tool we use to remote access the PCs without logging out the current user.

    Download the latest version of NoMachine and go through the install process. Reboot when prompted to restart. To check the service is running open the windows start menu and search for nomachine service. Nomachine service will automatically run at startup and should broadcast the pc name and ip to other pcs with nomachine installed on the same network. Ideally the PC should have a static IP Address though.

    1.1.1.1 Setting a static IP Address

    The IP Management Portal is where we request new static IP Addresses.

    If you don't have an account on the management portal, you can contact an admin to get an account or reserve an ip address for you. To reserve an IP Address click the Request IP button on the dashboard, and select a Range from the list, stating your purpose in the input field. A html alert will tell you if you're successful or not, if unsuccessful try a different range.

    After reserving an IP Address you have to change the IP Address on the PC itself.

    1. Open the control panel and click on network and internet.
    2. Click on Networks and Sharing Center
    3. Open the change adapter settings link in the left navbar, right-click the active ethernet adapter and open the properties.
    4. open the ipv4 settings and change the IP Address to your reserved IP.
    5. Change the subnet mask to 255.255.248.0 and the gateway to 10.211.128.1
    6. Change the main DNS server to 8.8.8.8 and the backup to 8.8.4.4
    7. Save settings and close.

    1.1.2 Setting up Avatar

    The avatar video we use is located in the AutomationTools directory on the nas drive

    //10.211.128.208/shared_area/AutomationTools/source_pc/avatar_60fps.mkv

    For the source_pc rest api to be able to play this video, it should be copied into the user's Videos folder. C:/Users/BBOX/Videos

    VLC link

    VLC will be needed to play the video automatically. Download the latest installer from this webpage and open the file, following the instructions given.

    Another thing to keep in mind is that vlc must be added to PATH first. To add vlc to PATH...

    1. Open windows file explorer, right-click on "This PC" and select properties. Navigate to Advanced System Settings.
    2. From the Advanced System Settings, click on Environment Variables.
    3. Under System Variables, locate and select Path and click Edit.
    4. Add a new Path and enter the location for vlc.exe. By default, this is usually C:/Program Files/VideoLAN/VLC/

    1.1.3 Setting up nircmd and multimonitortool

    NirCmd link

    MultiMonitorTool link

    Download the zip files from the above links. Create a folder called nirsoft in the C:/ directory and place nircmd.exe and multimonitortool.exe executables from the downloaded zip files inside.

    1.1.4 Setting up source pc rest api on new PC

    To set up the source pc rest api on a new source pc you will need the following dependencies...

    • nircmd
    • vlc
    • python 3.7 (or above)
    • Flask (installed using pip)
    • Tortoise SVN (to update the rest api when changes are made)

    1.1.4.1 Installing python

    Download the python installer from the link above and run the installer. Follow the on-screen instructions and restart the system if needed.

    You can check if you have python installed by typing python into the command line. If you enter the python interpreter, python is installed on your machine and you can close the interpreter by typing exit() into the command line.

    1.1.4.2 Installing the source pc automation rest server

    The url for the source pc automation folder is as follows - svn://10.211.128.12/automation/trunk/ApplianceAutomation/source_pc_automation.

    Download the TortoiseSVN installer from the following link and follow the installers steps to install. After you've installed TortoiseSVN, create a folder in the user's Documents folder called source_pc_automation. Right-click on this folder and select the svn checkout option.

    You can then input the svn url shown above and enter your credentials. The files automation_server.bat, automation_server.vbs, server.py and setup.bat should be added to the folder. Run the setup.bat file to start the python virtual environment and install Flask.

    You should now be able to run the automation_server.bat file to start the server. The pc will need nircmd, MultiMonitorTool and VLC installed to run avatar and change the monitor settings.

    1.1.4.3 Running on startup

    The python server is running on startup on all the PC's in the testrig at the moment.

    Steps to add server.py to windows startup...

    1. Open windows Run program and enter shell:startup
    2. Open the source pc automation folder and copy the automation_server.bat and the automation_vlc.bat files into the startup folder.
    3. You can also check if the app launches correctly by running automation_server.bat. The output should look like this...

    1.2 Video Automation

    To begin with the automation of Video testing will be the focus of the Source PC Automation Server. The following functionality will be required

    • Resolution change on multiple monitors
    • Sync loss on multiple monitors
    • Sync enable on multiple monitors
    • Read current resolution for a given monitor
    • Dump report on current monitors for traceability
    • Enable video test application on a particular monitor

    A number of different tools will be needed to facilitate the above requirements.

    1.2.1 Resolution Change

    The tool NirCmd from NirSoft can be used to set the resolution of any connected monitor. Below are the full options for the setdisplayoption

    Info

    setdisplay {monitor:index/name} [width] [height] [color bits] {refresh rate} {-updatereg} {-allusers}

    Changes your display settings. The [width] and [height] parameters represents the number of pixels on your screen. The [color bits] parameter represents the number of colors shown on your screen (8 - 256 color, 16 - 16bit color, 24 - 24bit color, and so on). {refresh rate} is an optional parameter that specifies the monitor refresh rate. If you specify the {-updatereg} parameter, the new settings will be saved in the Registry. If you specify both {-updatereg} and {-allusers} parameters, the new settings will be saved in the Registry for all users. If you have multiple monitors, you can use the optional monitor parameter, which specifies for which monitor you want to change the display settings. You can specify the monitor by index (0 for the first monitor, 1 for the second one, and so on) or by specifying a string in the system monitor name. The monitor name can be found in the Device manager of Windows: Right click on the monitor item, and then choose 'Properties'. The string displayed in the 'location' field is the monitor name. Example: 

    setdisplay 800 600 24 -updatereg
setdisplay 1024 768 24 90
setdisplay 1024 768 8
setdisplay monitor:1 1024 768 24 90
setdisplay monitor:name1 1024 768 24 90

    As an example of our use case we can set the resolution of the primary monitor to 3840x2160@60Hz with the following command

    nircmd.exe setdisplay 3840 2160 32 60

    1.2.2 Sync Loss On/Off

    The tool MultiMonitorTool from NirSoft can be used to enable/disable any monitor connected to the PC with the following command

    #Disable first monitor
MultiMonitorTool.exe /disable 1
#Disable second monitor
MultiMonitorTool.exe /disable 2
#Disable both monitors
MultiMonitorTool.exe /disable 1 2
#Enable first monitor
MultiMonitorTool.exe /enable 1
#Enable second monitor
MultiMonitorTool.exe /enable 2
#Enable both monitors
MultiMonitorTool.exe /enable 1 2

    1.2.3 Read Current Monitor Configuration

    The tool MultiMonitorTool from NirSoft can be used to save the current Monitor configuration to a text file using the following command

    #Saves the current configuration into the same folder as the exe
MultiMonitorTool.exe /SaveConfig config.txt

    1.2.4 Run Avatar

    To launch avatar in fullscreen mode and on repeat, we can use VLC to run the below command to launch the video 

    vlc C:\Users\BBOX\Videos\avatar.mkv --fullscreen --loop --one-instance

    The --one-instance parameter ensures that there will only ever be one instance of vlc open at a time, meaning sending a get request to the /run_avatar multiple times won't open multiple avatar videos playing at the same time.

    1.2.5 Endpoints

    The URL for a request is http://{IP of PC}:5000/{endpoint}

    1.2.5.1 GET Requests

    1. /run_avatar - Runs the avatar video in fullscreen mode and on repeat from the Videos folder.
    2. /get_monitor_info - Retreives info about the connected monitors to each PC

    1.2.5.2 PUT Requests

    1. /set_resolution - Sets the display resolution for one head from the PC 
      {
    "head": "which video output to set resolution of.",
    "refresh": "refresh rate of monitor",
    "xres": "x resolution of video feed",
    "yres": "y resolution of video feed",
}
    2. /set_sync_loss - Turns off the monitor connected to the PC 
      {
    "monitor": "name of monitor to turn off",
}
    3. /set_sync_enable - Turns on the monitor connected to the PC 
      {
    "monitor": "name of monitor to turn on",
}
