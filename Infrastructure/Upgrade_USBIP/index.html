    1. Release process for USBIP

    This document describes the board types and file locations for the ASICS Awsudcip USBIP and the process for committing updates to the build when a new version of the USBIP is released.

    The USBIP refers to the FPGA based USB implementation which replaces the net2272 USB chip. Depending on the board, this USBIP may be integrated into the the standard FPGA system.hdf or it may be build into a separate image (artix_image.bin) for programming into an ARTIX FPGA chip during boot-up.

    1.1 Release considerations

    Depending on the scope of the USBIP changes (changes to the standalone Artix binaries only, or changes requiring new Integrated FPGA images to be built), different release update strategies may be chosen.

    1.1.1 Artix-only releases

    If a USBIP release only targets boards containing an ARTIX (and not the integrated FPGA boards), then a release of the ARTIX binary files alone may be appropriate.

    There are two variants of the ARTIX binary files; One for the Emerald 2K boards (located in "(buildroot)/boards/blackbox/emerald2k/encoder/artix_image/artix_fpga.bin") and one for Emerald 4KR2 boards (located in "../boot/hdf/artix_fpga.bin").

    The most recent ARTIX binary files from the FPGA team can be checked and then copied from the FPGA release folders with the commands:

    Do a dry run of the planned copy to avoid surprises
    # In the buildroot directory (e.g. /home/developer/Buildroot_emerald2k_encoder/buildroot-2017.02.4)
./run copy-fpga-release --dry-run --devices ARTIX2K ARTIX4K
    Copy the files after verifying the dry run
    # In the buildroot directory (e.g. /home/developer/Buildroot_emerald2k_encoder/buildroot-2017.02.4)
./run copy-fpga-release --devices ARTIX2K ARTIX4K
    Verify the files are as expected and the metadata is consistent
    # In the buildroot directory (e.g. /home/developer/Buildroot_emerald2k_encoder/buildroot-2017.02.4)
./run copy-fpga-release --status --devices ARTIX2K ARTIX4K

    After verifying the files are as expected (using the copy-fpga-release tool), running a build to include the new binaries, and upgrading and checking the relevant boards, the updated Artix binaries can be committed to svn directly from buildroot with svn commit (i.e. without using arcanist). They are opaque binary files and can't be reviewed. A commit message with the release version and/or BUG number is recommended.

    1.1.2 Integrated FPGA Releases

    If a USBIP release targets both the ARTIX binaries and the integrated FPGA binaries, then it is usual to update the system.hdf and ARTIX binary files on all boards to the latest released FPGA version.

    The current buildroot version and the FPGA versions released by the FPGA team can be examined with the copy-fpga-release tool below. The most recent versions from the FPGA team can be copied to buildroot with the copy-fpga-release tool also.

    Verify the files are as expected and the metadata is consistent
    ./run copy-fpga-release --status
    Do a dry run to avoid surprises
    ./run copy-fpga-release --dry-run
    Copy the files for all devices (defaults to all boards) from FPGA release folders to buildroot
    ./run copy-fpga-release

    After verifying the files are as expected (using the copy-fpga-release tool), running a build to include the new binaries, and upgrading and checking the relevant boards; the updated system.hdf binaries, revision files and Artix binaries can be committed to svn directly from buildroot with svn commit (i.e. without using arcanist). The hdf and Artix files are opaque binary files and can't be reviewed. A commit message with the release version and/or BUG number is recommended.

    1.2 Boards and USBIP Types

    The following boards have the USBIP, either integrated or as an add-on file (June 2023):

    Board Short name Artix bin file Integrated into FPGA
    Emerald PE, DisplayPort, TX, Dual head TX_PE_DP_2H ✓ 2K
    Emerald PE, DisplayPort, TX, Single head TX_PE_DP_1H ✓ 2K
    Emerald SE, DisplayPort, TX, Dual head TX_SE_DP_2H ✓ 2K
    Emerald SE, DisplayPort, TX, Single head TX_SE_DP_1H ✓
    Emerald PER2, DVI, TX, Dual head TX_PER2_DVI_2H ✓ 2K
    Emerald PER2, DVI, TX, Single head TX_PER2_DVI_1H ✓ 2K
    Emerald SER2, DVI, TX, Single head TX_SER2_DVI_1H ✓
    Emerald 4K R2 TX_4KR2 ✓ 4K

    1.3 copy-fpga-release tool

    1.3.1 Checking current FPGA/USBIP information

    During the build process, the USBIP files are taken from the buildroot folders and incorporated into the upgrade image.

    The ./run copy-fpga-release --status command lists information about the system.hdf and artix bin files from both the build directory and the versions released by the FPGA team in the FPGA_bit_file folder.

    This command performs some validation checks on the binary files (system.hdf, artix_fpga.bin) and revision files ([Encoder|Decoder]_hdf_rev.txt) to verify that md5 checksums are as expected, that platform codes match the expected device and that folder names (where available) match the subsystem revisions.

    If the tool discovers inconsistencies during validation, it displays the suspicious value in *red preceded with asterisk.
