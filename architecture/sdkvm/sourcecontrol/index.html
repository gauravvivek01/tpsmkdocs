    1. Deskvue Source Control

    1.1 Source code organization

    SDKVM WORK Repo is the main repository; it contains all build and packaging related code. The core components of SDKVM such as libsystem, libmodel, logging, restapi, OSD, clients, workspace server are maintained in its own seperate GIT repository. These repositories are linked with the SDKVM WORK repo as a git submodule. Please refer git submodule foreach --recursive 'git rev-parse HEAD' for submodule details.

    SDKVM WORK REPO [Main repo contains ]

    |

    -- KVM CLIENT

    |

    -- RDP CLIENT

    |

    -- PCOIP CLIENT

    |

    -- OSD FLUTTER APP

    |

    -- WORKSPACE SERVER

    |

    -- SDKVM CONFIG/SCRIPTS

    |

    -- Discovery

    |

    -- Webservice

    |

    -- Librestapi

    |

    -- Open Source Libs

    |

    -- Customized Libs

    |

    -- BBConstants

    |

    -- Deskvue Upgrade

    |

    -- OSD Helper

    1.2 Source checkout and update

    • git clone --recurse-submodules -j8 git@github.com:blackbox-common-apps/SDKVM_WORKREPO.git
    • or for a branch git clone -b 1.1.0 --recurse-submodules -j8 git@github.com:blackbox-common-apps/SDKVM_WORKREPO.git

    1.2.1 To update all modules to the latest

    • git submodule update --remote

    1.2.2 To set the version number

    Edit the PROJECT_VERSION in build_component_jenkins.sh

    #!/bin/bash
PROJECT_VERSION="1.1.0"
DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
KVM_CLIENT="$DIR/kvm-client"

    1.2.3 To Commit code inside of a submodule checked out under SDKVM_WORKREPO

     git pull origin <Branch>
 git add <File to commit>
 git commit -m "Message"
 git push origin HEAD:<Branch>

    1.3 Add git submodule

    • git add submodule

    1.4 Build

    • To build the source code we use the build_component_jenkins.sh in SDKVM_WORKREPO to build all the components and package the firmware.
    • deskvue-docker-build repository

    1.5 Packaging

    The firmware will be packaged in the following packages

    • OPENBOX_BB_VERSION_amd64.deb [Open box custom package (patched version)]

    • DESKVUE_VERSION_amd64.deb [Contains all the proprietary software for Deskvue]

    • DESKVUE_VERSION.dkv [Un-encrypted Self extracting package for Deskvue software upgrade through OSD/Boxilla]

    • DESKVUE_VERSION.signed.dkv [Encrypted and signed self extracting package for Deskvue software upgrade through OSD/Boxilla]

    1.6 Commit of Code

    To commit inside of a submodule

     git pull origin main
 git add <File to commit>
 git commit -m "Message"
 git push origin HEAD:main

    1.7 Creating A Pull Request

    git checkout -b <new_branch>
    One may need to push the branch as well

    git push --set-upstream origin BUG-5633

    1.8 Cleaning up after a Pull Request on main

    git checkout main
git pull --rebase
# and then
git fetch origin --prune
# to remove your feature branch

    1.9 Cleaning up after a Pull Request on a branch

    git checkout <branch>
git pull --rebase
# and then
git fetch origin --prune
# to remove your feature branch

    1.10 Updating References in SDKVM_WORKREPO

    git config --file=.gitmodules submodule.<Module Name>.branch <Branch>
git submodule sync
git submodule update --init --recursive --remote

    For example:

    git config --file=.gitmodules submodule.workspaceserver.branch 1.2.0
git submodule sync
git submodule update --init --recursive --remote

    1.11 Identifying the Branch

    git branch --show-current

    1.12 Switching to a Branch

    git checkout  <new_branch>

    1.13 Tagging a release

    git tag <tagname> -a
git push origin --tags
    - Add release notes in the tag - Attach the release assets (fimrware) to new tag after downloading it from "latest" tag

    1.14 Using the same key pair on different machines

    Generate a key pair, for example:

    ssh-keygen -t ed25519 -C "your_name@blackbox.com"
    This will create two keys in ~/.ssh

     4 -r--------  1 developer developer   419 Jul  3 09:11 id_ed25519
 4 -rw-rw-r--  1 developer developer   110 Jul  3 09:12 id_ed25519.pub
    These keys can be copied to other machines from which you access the git repository.

    Add them to the ssh agent

    eval "$(ssh-agent -s)"
Agent pid 827391
ssh-add ~/.ssh/id_ed25519
    If you are copying these then you may need to change permissions after copying. Permissions 0664 for '/home/developer/.ssh/id_ed25519' are too open. It is required that your private key files are NOT accessible by others. This private key will be ignored.

    To fix this:

    chmod 400 ~/.ssh/id_ed25519

    Then try adding again:

    ssh-add ~/.ssh/id_ed25519

    1.16 Downloading Debian Packages from Github

    Builds can be found in https://github.com/blackbox-common-apps/SDKVM_WORKREPO/releases/tag/latest

    Branch Builds can be found in https://github.com/blackbox-common-apps/SDKVM_WORKREPO/releases/tag/latest-1.1.0

    Images are stored on the NAS in: 

    \\10.211.128.208\tftpboot\work\Deskvue_Release\

For example:

\\10.211.128.208\tftpboot\work\Deskvue_Release\V1.0.2\DESKVUE_1.0.2.r62.signed.dkz
