    Disabling the Client Menu

    • In the rc.xml file, we can remove the mousebind for the right mouse button on the titlebar. It has an associated action to show a client menu. This is the menu that displays when a title bar is right clicked.
    • In the rc.xml file, we should also remove the mousebind for the right mouse button associated with the bottom of the window. It has an associated action to show a client menu. This is the menu that displays when a title bar is right clicked.
    • Ideally all bindings for client-menu should be disabled.

    Openbox Utilities

    There are a number of utilities which can be used to configure OpenBox:

    Obconf

    ObConf installs new themes which use the .obt Openbox theme archive format for distribution. Open any .obt theme archive in your file browser, or use the interface provided inside ObConf. You can also use ObConf to create .obt Openbox theme archives for distributing your own themes.

    obmenu

    Obmenu is a menu editor designed for openbox. It's easy to use, allowing to get the most out of the powerful Openbox menu system, while hiding the xml layout from the user. It also (optionally) installs a bunch of dynamic menus (pipe menus), such as Gnome menus or a quick-navigator. You can also use the obxml module to easily write pipe menus of your own in Python.

    #!/bin/sh
if test -n "$1"; then
    echo "Syntax: openbox-session"
    echo
    echo "See the openbox-session(1) manpage for help."
  exit
fi
# Clean up after GDM
xprop -root -remove _NET_NUMBER_OF_DESKTOPS \
      -remove _NET_DESKTOP_NAMES \
      -remove _NET_CURRENT_DESKTOP 2> /dev/null
# Set up the environment
A="/usr/local/etc/xdg/openbox/environment"
test -r $A && . $A
A="${XDG_CONFIG_HOME:-"$HOME/.config"}/openbox/environment"
test -r $A && . $A

# Run Openbox, and have it run the autostart stuff
exec /usr/local/bin/openbox --startup "/usr/local/libexec/openbox-autostart OPENBOX" "$@"

    Controlling Power Saving and Monitor Sleep

    Useful Commands

    • xset s off Disable screen saver blanking
    • xset s 3600 3600 Change blank time to 1 hour
    • xset -dpms Turn off DPMS
    • xset s off -dpms Disable DPMS and prevent screen from blanking
    • xset dpms force off Turn off screen immediately
    • xset dpms force standby Standby screen
    • xset dpms force suspend Suspend screen
    • gsettings set org.gnome.settings-daemon.plugins.power sleep-inactive-ac-type 'nothing'
    • gsettings set org.gnome.settings-daemon.plugins.power sleep-inactive-battery-type 'suspend'

    To disable system Hibernate we can run the following:

    # To disable
sudo systemctl mask sleep.target suspend.target hibernate.target hybrid-sleep.target

# To check
sudo systemctl status sleep.target suspend.target hibernate.target hybrid-sleep.target

# To re-enable
sudo systemctl unmask sleep.target suspend.target hibernate.target hybrid-sleep.target

These will create the following:

Created symlink /etc/systemd/system/sleep.target  /dev/null.
Created symlink /etc/systemd/system/suspend.target  /dev/null.
Created symlink /etc/systemd/system/hibernate.target  /dev/null.
Created symlink /etc/systemd/system/hybrid-sleep.target  /dev/null.

    Booting without a Display Manager

    • Reference

    • Create the following file: /etc/systemd/system/getty@tty1.service.d/override.conf

    • Add these contents:

    [Service]
Type=simple
ExecStart=
ExecStart=-/sbin/agetty --autologin snuc --noclear %I 38400 linux
    - Add the following to the snuc .profile file in /home/snuc.profile: 

    #Startx Automatically
if [[ -z "$DISPLAY" ]] && [[ $(tty) = /dev/tty1 ]]; then

    sudo update-alternatives --config x-session-manager

    sudo systemctl disable lightdm

    sudo dpkg-reconfigure lightdm

    Network Configuration

    nmcli

    Allowing non-root user to run nmcli

    -Setting a static ipaddress

    chmod u+s,a-w /usr/bin/nmcli

    To reverse this we can run

    chmod u-s,a-w /usr/bin/nmcli

    Alowing non-route user to run the route command

    Note: It would be preferable to do any routing changes using nmcli

    sudo chmod u+s,a+w /usr/bin/route

    Allowing non-root user to run nmcli along with write permissions

    chmod u+s,a+w /usr/bin/nmcli

    Examples

    • Reference and Examples
    nmcli con add type ethernet con-name 'static-ip' ifname enp86s0 ipv4.method manual ipv4.addresses 192.168.1.98/24 gw4 192.168.1.254
nmcli con up id 'static-ip' #check IP is setup
nmcli connection show # show available connections
nmcli connection delete static-ip # delete a connection
nmcli con mod static-ip ipv4.dns 182.168.1.254 # add dns
nmcli con up id 'static-ip' # activate the static connection
nmcli con add type ethernet con-name 'dhcp' ifname enp86s0 ipv4.method auto
nmcli d show |  grep GATEWAY # show gateway settings
nmcli d show # Show all settings
nmcli connection modify dhcp ipv4.routes "10.211.128.192/21" # Add a route for a given connection, connection must be activated with nmcli con up id 'dhcp'
nmcli connection show dhcp | grep ipv4.routes # show routes for this connection
nmcli con show --active # show active connections
nmcli dev show | grep GENERAL.HWADDR # command to get a list of all the mac addresses 
nmcli radio wifi off # turn off wifi

    Delete all connections

    nmcli --terse connection show | cut -d : -f 1 | \
  while read name; do nmcli connection delete "$name"; done
