    1.2 Configuring OpenBox

    There are two principle files used to configure Openbox's start up sequence. The openbox-sessiosn file and the openbox.desktop file.

    1.2.1 OpenBox Session File

    Openbox uses a xession file which is located in /usr/local/bin/openbox-session 

    #!/bin/sh
if test -n "$1"; then
    echo "Syntax: openbox-session"
    echo
    echo "See the openbox-session(1) manpage for help."
  exit
fi
# Clean up after GDM
xprop -root -remove _NET_NUMBER_OF_DESKTOPS \
      -remove _NET_DESKTOP_NAMES \
      -remove _NET_CURRENT_DESKTOP 2> /dev/null
# Set up the environment
A="/usr/local/etc/xdg/openbox/environment"
test -r $A && . $A
A="${XDG_CONFIG_HOME:-"$HOME/.config"}/openbox/environment"
test -r $A && . $A

# Run Openbox, and have it run the autostart stuff
exec /usr/local/bin/openbox --startup "/usr/local/libexec/openbox-autostart OPENBOX" "$@"

    1.2.1.1 Turning on Logging in OpenBox

    • The Openbox log file can be found in: /home/snuc/.cache/openbox/openbox.log
    • For more verbose output change: 
    exec /usr/local/bin/openbox --startup "/usr/local/libexec/openbox-autostart OPENBOX" "$@"

    to

    exec /usr/local/bin/openbox --debug --startup "/usr/local/libexec/openbox-autostart OPENBOX" "$@"

    1.2.2 OpenBox Desktop File

    This is located in /usr/share/xsessions/openbox.desktop 

    [Desktop Entry]
Name=Openbox
Comment=Log in using the Openbox window manager (without a session manager)
Exec=/usr/bin/openbox-session
TryExec=/usr/bin/openbox-session
Icon=openbox
Type=Application

    1.2.2.1 Removing Other Boot Options

    To prevent other desktop and session options we remove any other desktop files in /usr/share/xsessions. openbox.desktop should be the only file present.

    1.2.3 Startup Sequence

    The trace below shows a typical openbox startup sequence

   2216 ?        Ss     0:00 /usr/bin/ssh-agent /usr/bin/im-launch /usr/local/bin/openbox-session

    1.3 Starting other applications from Openbox

    When Openbox begins to run it will look for an autostart script in /home/{USER}/.config/openbox.

    This file determines what programs will be start as Openbox launches.

    #!/bin/bash
logger "disabling Virtual Terminal Switching"
setxkbmap -option srvrkeys:none
logger "Setting session ID"
echo "0" > /tmp/session_id
# Start the workspace server
logger "Openbox autostart: Running workspaceserver from openbox" 
/usr/bin/startserver.sh
# Start Discovery
logger "Openbox autostart: Starting Discovery service"
/usr/bin/startdiscovery.sh
# Start Webservice
logger "Openbox autostart: Starting Webservice service"
/usr/bin/startwebservice.sh
# Configuring Keyboard
logger "Openbox autostart: Configuring Keyboard to gb local"
/usr/bin/setxkbmap -layout gb
sleep 2
logger "Openbox autostart: Starting deskvue application"
/home/snuc/.config/openbox/configure_monitors.sh
sleep 1
logger "Starting Compton Compositor"
compton --backend glx  --vsync opengl-swc &
logger "OSD Post Connect: Adding background images using feh" 
feh --bg-scale /home/snuc/.config/openbox/background.png &
/home/snuc/.config/openbox/disable_sleep.sh
logger "OSD Post Connect: Setting Keyboard layout"
setxkbmap -layout gb
logger "OSD Post Connect: Positioning the mouse for login"
xdotool mousemove 960 540
/home/snuc/SDKVM/emeralddeskvue/build/linux/x64/release/bundle/emeralddeskvue &

    1.13 Syslog Rotation

    There is a file at: /usr/lib/systemd/system/logrotate.timer this is also configured for daily

    I think this is the one that is meant to run, but may not be running

    The service can be enabled with:

    systemctl reenable logrotate.timer

    Logrotates configuration information can generally be found in two places on Ubuntu:

    • /etc/logrotate.conf: this file contains some default settings and sets up rotation for a few logs that are not owned by any system packages. It also uses an include statement to pull in configuration from any file in the /etc/logrotate.d directory.
    • /etc/logrotate.d/: this is where any packages you install that need help with log rotation will place their Logrotate configuration. On a standard install you should already have files here for core system tools like apt, dpkg, rsyslog and so on.

    To test

    sudo logrotate /etc/logrotate.conf --debug

    To check the service logs

    journalctl -u logrotate.service

# and the timer

journalctl -u logrotate.timer

    To check the service:

    systemctl status logrotate.timer
logrotate /etc/logrotate.d/rsyslog --debug
    The configuration for the Log Rotate Timer is in: /usr/lib/systemd/system/logrotate.timer

    [Unit]
Description=Daily rotation of log files
Documentation=man:logrotate(8) man:logrotate.conf(5)

[Timer]
OnCalendar=daily
AccuracySec=1h
Persistent=true

[Install]
WantedBy=timers.target

    1.13.1 To view the logrotate status and determine when the last rotation occured

    sudo cat /var/lib/logrotate/status

    1.13.2 Creating an Alternative Syslog Rotation Service

    Create two files in the directory /etc/systemd/system:

    altrotate.service

    [Unit]
Description=Alt Log Rotate

[Service]
Type=oneshot
ExecStart=/usr/bin/alt_logrotate.sh
    And

    altrotate.time

    [Unit]
Description=Alternative Logrotate monitoring

[Timer]
OnBootSec=5m
OnUnitActiveSec=1d

[Install]
WantedBy=timers.target

    Place the following file in /usr/bin

    alt_logrotate.sh

    #!/bin/bash
logger "Alr Log Rotate starting"
/usr/sbin/logrotate /etc/logrotate.conf
logger "Alr Log Rotate Complete"

    Enable the service

    The system information section has the main data of interest.

    To show the serial number:

    /usr/bin/sudo /usr/sbin/dmidecode -s system-serial-number

    1.14.1 SimplyNuc Serial Number Format

    • Production serial numbers that shipped recently are in this format S1L1T70023M0001
    • The first 8 digits of the serial number are the Model number, ie S1L1T700 remaining 7 digits are yy-m-xxxx (where xxxx is a sequential number).
    • The months are as follows: Y Jan, F Feb, H Mar, L Apr, M May, E Jun, J Jul, A Aug, S Sep, O Oct, N Nov, D Dec.

    Model number will not change (S1L1T700) for future production shipments, this is our custom configuration so this is the S/N format going forward.
